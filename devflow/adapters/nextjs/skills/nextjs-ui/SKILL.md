## shadcn/ui — copy-paste model

shadcn/ui is NOT a traditional npm dependency — components live in `components/ui/` as your own code.

- Add component: `npx shadcn@latest add [component]`
- Customize: edit `components/ui/[component].tsx` directly — it's your code
- Update: re-run `npx shadcn@latest add [component]` (overwrite), not npm
- Common components: `Button`, `Input`, `Label`, `Select`, `Dialog`, `Sheet`, `Popover`, `DropdownMenu`, `Card`, `Badge`, `Toast`, `Form`

## `cn()` utility — merge className

```ts
// lib/utils.ts (generated by shadcn setup)
import { clsx, type ClassValue } from 'clsx'
import { twMerge } from 'tailwind-merge'

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}

// Usage:
className={cn(
  'base-classes rounded-md border',
  isActive && 'bg-primary text-primary-foreground',
  props.className  // always accept className for composability
)}
```

## Design tokens — CSS variables

- Tokens defined in `globals.css` as CSS custom properties
- Tailwind consumes via `tailwind.config.ts`
- Core tokens (shadcn defaults): `--background`, `--foreground`, `--primary`, `--primary-foreground`, `--secondary`, `--muted`, `--accent`, `--border`, `--ring`, `--radius`
- Always use semantic tokens, never hardcoded colors:

```tsx
// ✅ Correct
className="bg-background text-foreground border-border"

// ❌ Wrong
className="bg-white text-gray-900"
style={{ backgroundColor: '#fff' }}
```

## Dark mode — next-themes

Setup in `providers.tsx`:

```tsx
'use client'
import { ThemeProvider } from 'next-themes'

export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <ThemeProvider
      attribute="class"
      defaultTheme="system"
      enableSystem
      disableTransitionOnChange
    >
      {children}
    </ThemeProvider>
  )
}
```

Toggle in Client Component:

```tsx
'use client'
import { useTheme } from 'next-themes'

function ThemeToggle() {
  const { theme, setTheme } = useTheme()
  return (
    <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
      Toggle
    </button>
  )
}
```

- Root layout: add `suppressHydrationWarning` on `<html>` — prevents hydration mismatch
- Dark mode parity: every interactive state (hover, focus, active, disabled) must have dark variant

## Responsive layout

- Tailwind breakpoints: `sm` (640px), `md` (768px), `lg` (1024px), `xl` (1280px), `2xl` (1536px)
- Mobile-first: default = mobile, `sm:`/`md:`/`lg:` = override for larger screens
- Container: `<div className="container mx-auto px-4 md:px-8">`
- Grid: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4`
- Flex: `flex flex-col md:flex-row items-start md:items-center gap-4`

## Icons — lucide-react

- Included with shadcn/ui setup
- Import: `import { Search, Settings } from 'lucide-react'`
- Size: always `className="h-4 w-4"` or Tailwind variants — no inline style

## Typography

- Headings: `text-2xl font-bold tracking-tight`
- Body: `text-sm text-muted-foreground`
- Rich text: `@tailwindcss/typography` plugin → `prose` class

## Anti-patterns

- Hex/rgb inline (`style={{ color: '#333' }}`)
- Overriding Radix `aria-*` props without valid reason
- Not accepting `className` prop in custom components — breaks composability
- Removing `disableTransitionOnChange` from ThemeProvider — causes flash

## Review checklist

- [ ] No hex/rgb hardcoded — CSS variable tokens only
- [ ] Dark mode parity on all interactive states
- [ ] Responsive: mobile-first, Tailwind breakpoints
- [ ] `cn()` used for className merge
- [ ] `suppressHydrationWarning` on `<html>` in root layout
- [ ] Icon size from Tailwind classes, not inline style
