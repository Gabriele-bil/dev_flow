# DevFlow Plugin — Claude Code Context

Claude Code + Cursor plugin for spec-driven development pipelines.
Read `templates/devflow/CONTRIBUTING.md` for quality bar and style guidelines before modifying any skill.

## Critical structure

- `templates/devflow/` — plugin source (edit here)
- `dist/devflow/` — installable plugin (build output, committed)
- `dist/devflow/.claude-plugin/plugin.json` — Claude Code manifest (generated)
- `templates/devflow/skills/<name>/SKILL.md` — core pipeline skills
- `templates/devflow/adapters/<stack>/skills/<name>/SKILL.md` — adapter-specific skills
- `templates/devflow/hooks/` — shell scripts + `hooks.json` (lifecycle hooks)
- `templates/devflow/commands/` — slash command entry points

## Operational rules

- Every SKILL.md must have YAML frontmatter with `name` and `description`
- Required sections: `## Purpose`, `## Core Principles`, `## When NOT to Use`, `## I/O Reference`
- After any SKILL.md change: validate with `bash templates/devflow/scripts/validate-skills.sh --strict`
- After any core skill description/trigger change: run `bash templates/devflow/scripts/run-evals.sh` (checks NL trigger routing and description collisions — see `templates/devflow/evals/README.md`)
- After any change to skills/adapters/hooks: rebuild with `bash scripts/build-plugin.sh`
- Do not touch `templates/devflow/config.md` — generated by `devflow.setup` in the consumer project
- Do not write files inside `devflow/features/` — those are consumer project artefacts

## Useful commands

```bash
# Validate all SKILL.md files
bash templates/devflow/scripts/validate-skills.sh --strict

# Run Tier-2 trigger/routing + description-collision evals
bash templates/devflow/scripts/run-evals.sh

# Rebuild dist/ after changes
bash scripts/build-plugin.sh

# Check hook script syntax
bash -n templates/devflow/hooks/<script>.sh
```

## What NOT to do

- Do not edit files in `dist/` directly — they are build output
- Do not add business logic to templates — it belongs in skills
- Do not add synchronous heavy hooks (`async: true` + `timeout` for I/O operations)
- Do not modify `hooks.json` without updating the corresponding scripts in `templates/devflow/hooks/`
